# 📱 영숙 — 회사 목표, 각 에이전트의 개인 목표, 최근 의사결정, 메모리를 검토하여 현재 가장 가치 있는 단일 작업을 결정하고, 해당 작업의 실행 계획을 수립하라.

📱 **업무 수행 계획 (Task Plan)**

현재 귀하의 요청은 시스템의 **업무 수행 계획(Task Plan)**을 요청하는 것으로 이해됩니다. 이는 특정 목표나 작업을 수행하기 위한 단계적이고 구조화된 접근 방식을 의미합니다.

현재 제공된 정보는 **명령**에 가깝기 때문에, 저는 이 요청에 대해 다음과 같이 **현재 상황 분석 및 다음 단계 제안**을 수행하겠습니다.

### 1. 현재 상황 분석 (Current Situation Analysis)

*   **요청의 성격:** 시스템의 내부 프로세스(Task Plan) 요청.
*   **제공된 정보:** 이전 대화 맥락(Context)에 기반하여, 사용자가 현재 어떤 작업을 시작하기를 원하는지에 대한 명확한 목표는 제시되지 않았습니다.
*   **핵심 제약 조건:** 저는 제공된 컨텍스트 내에서만 추론하고 응답해야 합니다.

### 2. 다음 단계 제안 (Proposed Next Steps)

어떤 종류의 'Task Plan'을 원하시는지 명확히 하기 위해, 다음 중 원하는 것을 선택해 주시거나 구체적인 목표를 제시해 주시면 맞춤형 계획을 수립해 드릴 수 있습니다.

1.  **프로젝트/목표 설정:** 수행하고자 하는 구체적인 목표(예: 블로그 포스팅 작성, 코드 디버깅, 시장 조사 보고서 작성 등)를 제시해 주세요.
2.  **기존 정보 기반 계획:** 현재까지 논의된 내용(예: 이전 대화에서 언급된 '회사 운영' 또는 '콘텐츠 제작' 관련)을 바탕으로 다음 단계의 로드맵을 제시해 주세요.
3.  **역할 기반 계획:** 특정 역할(예: 콘텐츠 전략가, 마케터, 개발자)을 부여하고 그 역할에 맞는 계획을 요청해 주세요.
4.  **일반 프로세스 요청:** 목표가 정해지지 않았다면, 일반적인 프로젝트 관리 프로세스(기획 → 실행 → 검토 → 완료)를 요청해 주세요.

**어떤 방향으로 Task Plan을 진행하면 될까요? 구체적인 목표를 말씀해 주시면 즉시 계획을 시작하겠습니다.**
